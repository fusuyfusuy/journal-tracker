# .mcd/ — MCD-Flow artifacts

> **Status:** Historical reference. Cycle semantics still accurate, file paths are v0.1.0.

These files were generated by [MCD-Flow](https://github.com/) during the original single-package Bun + `bun:sqlite` implementation. The codebase has since been migrated to a NestJS monorepo, but the requirements, statechart, and manifest are still the source of truth for what the system *does* (its flows, constraints, error conditions, and fault-isolation guarantees).

## What's still correct

- [`requirements.md`](./requirements.md) — actors, flows, constraints, error conditions, out-of-scope items. All still apply.
- [`statechart.mmd`](./statechart.mmd) — per-check-cycle state machine. Transition names (`fetchJournalArticles`, `persistArticle`, `dispatchNotification`, `handleFetchError`, etc.) map 1:1 onto current implementation methods in `apps/worker/src/cycle/cycle.service.ts` and the fetcher/notifier providers.
- [`manifest.json`](./manifest.json) — entity shapes, feature list, transitions. Entities match the TypeORM classes in `libs/database/src/entities/`.

## What drifted

- **File paths** in the manifest and registry (`src/db.ts`, `src/fetchers/rss.ts`, …) reflect the v0.1.0 single-package layout. Current equivalents:
  - `src/db.ts` → `libs/database/src/`
  - `src/fetchers/*` → `apps/worker/src/fetchers/*.fetcher.ts`
  - `src/notifiers/*` → `apps/worker/src/notifiers/*.notifier.ts`
  - `src/cycle.ts` → `apps/worker/src/cycle/cycle.service.ts` (19 action functions became methods on one service)
  - `src/scheduler.ts` → `apps/worker/src/cycle/scheduler.bootstrap.ts` (interval loop → BullMQ repeatable job)
  - `src/index.ts` → `apps/worker/src/main.ts` + `apps/api/src/main.ts`
- **Runtime**: `bun:sqlite` + raw SQL → TypeORM + `better-sqlite3`.
- **Scheduling**: in-process `setInterval` → BullMQ + Redis.
- **Deps**: Zod + `fast-xml-parser` → `class-validator` + `class-transformer` + `fast-xml-parser` (Zod no longer used).

## Reading the original code

The pre-migration source is preserved at tag [`v0.1.0-bun`](../../../tree/v0.1.0-bun). Checkout with:

```bash
git checkout v0.1.0-bun
```

## Extending via MCD-Flow

To regenerate this spec from the current monorepo, run [`/mcd-import`](https://github.com/). That produces a fresh manifest rooted in the Nest module tree; after that, `/mcd "<new feature>"` runs in diff mode and keeps the statechart aligned with the code.
